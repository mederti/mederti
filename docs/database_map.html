Triggers & Utility Functions

⚡
trg_drugs_search_vector  ·  BEFORE INSERT OR UPDATE ON drugs
Maintains three columns: generic_name_normalised (lower/trim), brand_names_text (array_to_string), and search_vector (weighted tsvector: A=generic, B=brand, C=atc+category). Runs update_drug_search_vector().
⚡
trg_shortage_events_id  ·  BEFORE INSERT ON shortage_events
Computes deterministic MD5 dedup key: MD5(drug_id | data_source_id | country_code | start_date). Only fires if shortage_id is NULL or empty — allows manual override in tests. Runs set_shortage_id().
⚡
trg_*_updated_at  ·  BEFORE UPDATE (5 tables)
Shared set_updated_at() function applied to: drugs, manufacturers, shortage_events, recalls, intelligence_sources. Sets updated_at = NOW() on every row update.
🔧
mark_stale_shortages()  ·  Utility function (SECURITY DEFINER)
Updates active/anticipated shortage_events where last_verified_at < NOW() - 7 days → status='stale'. Writes audit_log entries for every row changed. Returns row count. Called on a daily cron schedule via Supabase Edge Function scheduler.

Resilience Score Formula (computed, not stored)

score = 100
score -= count(recalls within last 12 months) × 5
score -= count(Class I recalls within last 24 months) × 15
score -= count(Class I recalls that caused/preceded a shortage) × 20
score = max(0, min(100, score))

Thresholds:  80–100 → green   60–79 → yellow   0–59 → red
